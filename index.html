Application Settings & Setup

Configure your cloud services and API keys to unlock the full potential of Unity.

🎨 Appearance

☁️ GitHub Cloud Integration

Purpose: Stores all uploaded CSV stats and calculations persistently in the cloud.

Cost: 100% Free

Setup Guide:

  1. Create a free account on GitHub.
  2. Create a new Public or Private repository.
  3. Go to Settings > Developer settings > Personal access tokens > Tokens (classic).
  4. Generate a new token with repo scope checked. Copy the token immediately.

🤖 Gemini AI Scanner

Purpose: Powers the OCR Screenshot Scanner (reading game images natively).

Cost: Free Tier (15 requests/min, 1 million reqs/day). Pay-as-you-go if exceeded (fractions of a cent per scan).

Setup Guide:

  1. Visit Google AI Studio and sign in.
  2. Click Create API Key and select (or create) a Google Cloud project.
  3. Copy the generated API Key.
  4. Note: If you constantly hit quota errors, ensure billing is set up on your Cloud Project (the Free tier requires a linked card to verify identity).

🔥 Firebase Cloud Sync

Purpose: Drives the "My Alliance" live shared roster and instant AI scan syncing.

Cost: Spark Plan is 100% Free (1GB storage, 10GB/mo transfer - highly sufficient for text data).

Setup Guide:

  1. Go to the Firebase Console and add a new project.
  2. In the left menu, select Realtime Database under Build.
  3. Click Create Database (start in Test Mode, or edit Rules to allow reads/writes).
  4. Copy the Database URL provided (e.g., https://your-project.firebaseio.com).
  5. Paste it below. No further API keys are needed!

🎯 Recruiting Database Sync

Purpose: Isolates Applicant Scanner data from the main Alliance roster.

Default URL: https://recruitingroster-default-rtdb.firebaseio.com/

Setup Guide:

  • This database acts as a staging area specifically for the Recruiting Tab.
  • It allows Kingdom leadership to review applicant scan data without cluttering the master `My Alliance` database.
  • Paste your dedicated recruiting database URL below.

☁️ AWS DynamoDB Sync

Purpose: A high-capacity alternative to Firebase. If toggled ON, all syncing diverts to AWS.

Cost: Free Tier includes 25GB Storage & 200M requests per month.

📅 Google Calendar Sync

Purpose: Drives the live Events tab schedule so players can subscribe via iOS/Android.

Cost: 100% Free

Setup Guide:

  1. Open Google Calendar and create a new Calendar for your Alliance.
  2. Hover over the new calendar in the left sidebar, click the three dots, and select Settings and sharing.
  3. Under Access permissions for events, check Make available to public.
  4. Scroll down to Integrate calendar and copy the Calendar ID (e.g., xyz@group.calendar.google.com). Paste it below.
  5. Go to the Google Cloud Console. Create a Project if you haven't already.
  6. Click + CREATE CREDENTIALS > API Key. Paste the generated key below.
  7. Finally, go to the API Library and click ENABLE for the Google Calendar API.

How to Read This Chart

Data Source: Using Overview Data (Start Scan vs End Scan differences).

Visualization (PCA) & Color Coding

Instead of a simple "KP vs Deads" chart, we use Principal Component Analysis (PCA) to analyze multiple metrics (Power, Troops, Kills, Deads) simultaneously. The chart clusters players based on combat behavior:

  • ● Heroes (Blue): High KP, Low Deads (Efficient).
  • ● Warriors (Green): High KP, High Deads (Heavy fighters).
  • ● Feeders (Red): Low KP, High Deads (Negative trades).
  • ● Slackers (Gray): Low KP, Low Deads (Inactive).

Understanding the Axes

The numbers represent Standard Deviations from the Kingdom Average (0).

  • X-Axis (Activity Volume):
    0 = Average. Positive (+10/20) = Whales/Grinders. Negative = Inactive.
  • Y-Axis (Efficiency):
    0 = Average style. Positive = High Cost (Many Deads). Negative = High Efficiency.
